How I Built the PE Value Creation Dashboard

A simulated Private Equity portfolio management tool that tracks value creation across multiple portfolio companies, with waterfall analysis, initiative tracking, and company-level drill-downs.

Try it live

The Idea

Working in product at an insurtech backed by PE gave me a front-row seat to how value creation plans are tracked in practice: lots of spreadsheets, disconnected reporting, and quarterly board slides. I wanted to build what a modern, purpose-built tool could look like.

The goal was a single dashboard where an operating partner could see the full portfolio at a glance, drill into any company, track the status of value creation initiatives, and understand where EBITDA growth was actually coming from. All with real chart types used in PE (waterfall bridges, IRR/MOIC cards, cohort sparklines).

Key Features

Portfolio overview

All portfolio companies in one view with IRR, MOIC, entry vs current EV, EBITDA growth, and holding period at a glance.

Value bridge

Waterfall chart breaking down total value creation by EBITDA growth, multiple expansion, and leverage paydown across the portfolio.

Initiatives tracker

Track value creation initiatives by company, category, owner, status, and projected EBITDA impact with progress bars.

Company drill-down

Tab-based navigation to open any portfolio company and see its revenue, EBITDA trend, initiatives, and financial summary in detail.

Multiple chart types

Area charts for revenue trends, bar charts for EBITDA by period, sparklines for compact KPI cards, and a custom waterfall chart component.

Tabbed workspace

Open multiple portfolio companies simultaneously as tabs, just like a Bloomberg terminal or analyst workspace.

Data Architecture

All data is simulated and typed. Four fictional portfolio companies (Healthcare, Technology, Industrials, Consumer) with realistic financial profiles, each with their own initiatives, financials, and value bridge components.

Data lives in TypeScript modules under /src/data/ with strongly typed interfaces covering companies, financials, initiatives, summaries, and value bridge entries. A custom usePortfolioData hook handles all lookups and aggregations so components stay clean.

The waterfall chart was the most interesting build challenge since Recharts doesn't have a native waterfall type. It's built on top of a stacked bar chart with transparent "spacer" segments to create the floating bar effect.

How It Would Work in Production

The typed dataset is a stand-in for what, in a real PE context, would come from several different sources depending on the firm's stack:

  • Portfolio company financials - pulled from ERP exports (NetSuite, SAP, Xero) or a central data warehouse that aggregates monthly management accounts across the portfolio
  • Value creation initiatives - typically owned in a portfolio management system like Dynamo, Allvue, or a custom Airtable/Notion setup; the dashboard would replace the spreadsheet layer on top of that
  • IRR and MOIC calculations - sourced from the fund's finance team or a deal management platform, not computed in the UI; the dashboard consumes and displays them
  • Data room feeds - for newer investments still in diligence, structured document stores (Datasite, Intralinks) expose APIs that could feed early-stage KPI snapshots

The typed data model deliberately mirrors what those real sources would return: each company has an ID, sector, financial time series, and a list of named initiatives with owners, status, and projected EBITDA impact. Swapping the static modules for API calls or a data layer (GraphQL, REST, or direct DB queries) would be the main production change - the component and hook architecture is already built to support it.

Design

Dark gray interface (gray-950 base) that feels at home on a Bloomberg terminal or Palantir dashboard. Teal accents for positive signals, amber for at-risk, red for off-track. Typography stays compact and data-dense by design.

The tab-based navigation was a deliberate UX choice to mirror how analysts actually work: multiple companies open at once, switching context without losing their place.

What I Learned

  • Building a custom waterfall chart in Recharts using stacked bars with transparent spacer segments
  • Designing data schemas for PE-style reporting: linking companies, initiatives, financials, and value bridge components in a consistent type hierarchy
  • Tab-based state management with dynamic company tabs that open, close, and restore active context
  • How to make data-dense dashboards readable: color coding, typography hierarchy, and consistent chart language
  • The gap between how PE firms track value creation today (spreadsheets) and what a proper product could look like
